Overview

WAHA Gateway menyediakan RESTful API yang powerful untuk mengintegrasikan WhatsApp ke dalam aplikasi Anda. API ini dirancang khusus untuk developer dengan dokumentasi lengkap dan contoh kode yang mudah dipahami.

Base URL

http://localhost:8000/api/v1

Fitur Utama

  • RESTful API yang mudah digunakan
  • Dukungan berbagai jenis pesan (text, image, video, document, poll, button, list)
  • Webhook real-time untuk pesan masuk
  • Multi-session management
  • Rate limiting untuk keamanan
  • Dokumentasi lengkap dengan contoh kode

Autentikasi

Semua request ke API memerlukan API Key yang dapat Anda buat di dashboard setelah mendaftar.

Cara Menggunakan API Key

Kirim API Key di header HTTP dengan format berikut:

X-Api-Key: waha_your_api_key_here
Penting: Jangan pernah membagikan API Key Anda kepada pihak lain. Simpan API Key dengan aman dan jangan commit ke repository publik.

Endpoints

Berikut adalah daftar endpoint utama yang tersedia:

GET /sessions

Mendapatkan daftar semua session WhatsApp yang terhubung.

POST /messages

Mengirim pesan WhatsApp. Mendukung berbagai jenis pesan.

GET /messages

Mendapatkan daftar pesan yang telah dikirim atau diterima.

POST /sessions/{session}/messages/sync

Sinkronisasi pesan masuk dari WAHA ke database.

Mengirim Pesan

Untuk mengirim pesan, gunakan endpoint POST /api/v1/messages dengan format berikut:

Format Request

{ "session_id": "session_xxx", "to": "6281234567890", "message_type": "text", "text": "Halo, ini pesan dari API!" }

Format Response

{ "success": true, "data": { "message_id": "uuid", "whatsapp_message_id": "3EB0...", "status": "sent", "to": "6281234567890" } }

Jenis Pesan yang Didukung

WAHA Gateway mendukung berbagai jenis pesan WhatsApp:

1. Text Message

Pesan teks biasa.

{ "message_type": "text", "text": "Pesan teks" }

2. Image Message

Mengirim gambar dengan URL.

{ "message_type": "image", "image": "https://example.com/image.jpg", "caption": "Caption gambar" }

3. Video Message

Mengirim video dengan URL.

{ "message_type": "video", "video": "https://example.com/video.mp4", "caption": "Caption video" }

4. Document Message

Mengirim dokumen (PDF, DOC, dll).

{ "message_type": "document", "document": "https://example.com/file.pdf" }

5. Poll Message

Mengirim polling interaktif.

{ "message_type": "poll", "poll_name": "Pertanyaan?", "poll_options": ["Opsi 1", "Opsi 2"] }

6. List Message

Mengirim pesan dengan format list interaktif.

{ "message_type": "list", "message": { "title": "Judul", "description": "Deskripsi", "button": "Pilih", "sections": [...] } }

Contoh Kode

PHP (cURL)

// Kirim pesan text $ch = curl_init('http://localhost:8000/api/v1/messages'); curl_setopt($ch, CURLOPT_RETURNTRANSFER, true); curl_setopt($ch, CURLOPT_POST, true); curl_setopt($ch, CURLOPT_HTTPHEADER, [ 'X-Api-Key: waha_your_key', 'Content-Type: application/json' ]); curl_setopt($ch, CURLOPT_POSTFIELDS, json_encode([ 'session_id' => 'session_xxx', 'to' => '6281234567890', 'message_type' => 'text', 'text' => 'Halo dari API!' ])); $response = curl_exec($ch); curl_close($ch);

JavaScript (Fetch)

// Kirim pesan text fetch('http://localhost:8000/api/v1/messages', { method: 'POST', headers: { 'X-Api-Key': 'waha_your_key', 'Content-Type': 'application/json' }, body: JSON.stringify({ session_id: 'session_xxx', to: '6281234567890', message_type: 'text', text: 'Halo dari API!' }) }) .then(response => response.json()) .then(data => console.log(data));

Python (Requests)

import requests # Kirim pesan text url = 'http://localhost:8000/api/v1/messages' headers = { 'X-Api-Key': 'waha_your_key', 'Content-Type': 'application/json' } data = { 'session_id': 'session_xxx', 'to': '6281234567890', 'message_type': 'text', 'text': 'Halo dari API!' } response = requests.post(url, json=data, headers=headers) print(response.json())

Webhook

Webhook memungkinkan Anda menerima notifikasi real-time ketika ada pesan masuk ke session WhatsApp Anda. Ini sangat berguna untuk membangun aplikasi yang reaktif dan responsif.

Konfigurasi Webhook

Untuk mengatur webhook, gunakan endpoint berikut:

POST /webhooks

Membuat webhook baru untuk menerima notifikasi pesan masuk.

Format Request

{ "url": "https://your-domain.com/webhook", "events": ["message", "status"], "session_id": "session_xxx" }

Format Webhook Payload

Ketika pesan masuk, webhook akan mengirim POST request ke URL yang Anda tentukan dengan payload berikut:

{ "event": "message", "session_id": "session_xxx", "data": { "from": "6281234567890@c.us", "to": "6289876543210@c.us", "body": "Pesan masuk", "message_type": "text", "timestamp": 1234567890 } }

Mengelola Webhook

GET /webhooks

Mendapatkan daftar semua webhook yang telah dibuat.

DELETE /webhooks/{id}

Menghapus webhook tertentu.

Tips
  • Pastikan URL webhook Anda dapat diakses dari internet (gunakan HTTPS untuk produksi)
  • Webhook akan otomatis retry jika endpoint Anda tidak merespons
  • Gunakan signature verification untuk memastikan request berasal dari WAHA Gateway

Template Management

Template memungkinkan Anda membuat pesan dengan variabel dinamis yang dapat digunakan berulang kali. Sangat berguna untuk pesan otomatis, notifikasi, dan kampanye marketing.

Membuat Template

Untuk membuat template baru, gunakan endpoint berikut:

POST /templates

Membuat template baru dengan variabel yang dapat diganti.

Format Request

{ "name": "Order Confirmation", "content": "Halo {{name}}, pesanan Anda #{{order_id}} telah dikonfirmasi. Total: Rp {{amount}}", "message_type": "text", "variables": ["name", "order_id", "amount"], "description": "Template untuk konfirmasi pesanan" }

Menggunakan Template

Untuk mengirim pesan menggunakan template, gunakan template_id dan variables dalam request:

{ "to": "6281234567890", "template_id": "template-uuid-here", "variables": { "name": "John Doe", "order_id": "ORD-12345", "amount": "150.000" } }

Mengelola Template

GET /templates

Mendapatkan daftar semua template yang telah dibuat.

GET /templates/{id}

Mendapatkan detail template tertentu.

PUT /templates/{id}

Memperbarui template yang sudah ada.

DELETE /templates/{id}

Menghapus template tertentu.

POST /templates/{id}/preview

Preview template dengan variabel yang sudah diisi.

Format Variabel

Variabel dalam template menggunakan format {{variable_name}}. Contoh:

"Halo {{name}}, terima kasih telah memesan. Total pembayaran: Rp {{total}}"
Tips
  • Nama variabel harus alphanumeric dan underscore (a-z, A-Z, 0-9, _)
  • Variabel akan otomatis diekstrak dari content jika tidak didefinisikan
  • Template dapat digunakan untuk berbagai jenis pesan (text, image, button, list)
  • Gunakan preview untuk melihat hasil sebelum mengirim
